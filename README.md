# Bookmarks Checker

Утилита для анализа закладок браузера Floorp (Firefox‑совместимый браузер) в системах Windows 10 и Manjaro.
Программа позволяет получить текстовый отчёт по структуре закладок и количеству закладок в выбранной папке и её подкатегориях, а также управлять конфигурацией и запуском через web‑интерфейс.

## Возможности

- Автоматический поиск профиля Floorp на Windows и Linux.
- Копирование базы закладок `places.sqlite` в локальную папку `data`.
- Подсчёт количества закладок и вложенных папок в выбранной директории (по умолчанию — `KDE Store`).
- Формирование текстового отчёта и сохранение его в папку `docs`.
- Очищение от файлов директории для формирования отчетов.
- Асинхронная работа с базой данных через `aiosqlite`.
- Web API (FastAPI) для настройки конфигурации и получения отчёта.

## Конфигурация и логика работы

1. **Конфигурация**

   Конфигурация задаётся в `src/bookmarks/config.py` через класс `Config`:

   - `host` — хост (по умолчанию `127.0.0.1`);
   - `port` — порт (по умолчанию `8000`);
   - `bookmarks_folder` — имя папки закладок, по которой строится отчёт (по умолчанию `KDE Store`);
   - `report_folder` — папка, куда сохраняется отчёт (по умолчанию `docs`);
   - `report_file` — базовое имя файла отчёта (по умолчанию `Bookmarks KDE Store`);
   - `database_file` — имя файла базы данных закладок (по умолчанию `places.sqlite`);
   - `browser` — имя браузера/директории (по умолчанию `Floorp`);
   - `custom_profile` — опциональный путь к профилю, если нужно указать конкретный;
   - `path_bookmarks` — вычисляемый путь к профилю браузера.

   В методе `__post_init__` программа определяет ОС и вычисляет путь к профилям Floorp:

   - Windows: `C:/Users/<user>/AppData/Roaming/Floorp/Profiles/...`
   - Manjaro: `/home/<user>/.floorp/...`

   Далее выбирается первый профиль, подходящий под маску `*.default-default`, если `custom_profile` не задан.

2. **Копирование базы и формирование отчёта**

   В `src/bookmarks/report.py`:

   - проверяется наличие исходного файла `places.sqlite` в профиле браузера;
   - файл копируется в локальную папку `data`;
   - создаётся экземпляр `BookmarksDatabase`, который:
     - открывает базу через `aiosqlite`,
     - ищет папку закладок с именем `bookmarks_folder`,
     - подсчитывает количество закладок и подкатегорий в ней и в её дочерних папках,
     - возвращает многострочный текстовый отчёт.

   Отчёт сохраняется в файл:

   ```text
   docs/<report_file>.txt
   ```

## Пример отчета

```text
Initial catalog: KDE Store
bookmarks: 0
catalogs: 2
---------------------------------------------------------------------------------------------
Глобальные темы
bookmarks: 3
---------------------------------------------------------------------------------------------
Заставки
bookmarks: 1
```

## Сценарии

### Сформировать и сохранить отчёт в текстовый файл

```bash
uv run start
```

При этом:

1. Определяется путь к профилю браузера Floorp на основе настроек `Config`.
2. Файл базы закладок `places.sqlite` копируется в папку `data` в текущем каталоге (если нужно).
3. Из копии базы формируется отчёт по закладкам.
4. Отчёт сохраняется в папку `docs` под именем `<report_file>.txt`.

По умолчанию результат будет сохранён в файл:

`docs/Bookmarks KDE Store.txt`

### Очистить от файлов директорию для формирования отчётов

```bash
uv run clear
```

При этом:

1. Определяется путь к папке отчетов (по умолчанию `docs`).
2. Проводится безопасная очистка этой директории.
3. Формируется ответ об успешности данной процедуры, выводится в терминал и сохраняется в журнал.

По умолчанию очищается от файлов директория `docs/`.

### Web API

Приложение предоставляет web‑интерфейс на FastAPI для настройки конфигурации и получения отчёта.

Запуск через команду:

#### Запуск web‑сервера

```bash
uv run web
```

По умолчанию сервер стартует на `http://127.0.0.1:8000`

#### Документация API

После запуска перейдите в браузере:

* Swagger UI: `host:port/docs#`

#### Основные эндпоинты

##### 1. Задать конфигурацию

`PUT /config`

Эндпоинт принимает данные формы и обновляет конфигурацию утилиты.

Тело запроса (form-data):

* `📜 Установить значения по умолчанию` (`is_default`), **опционально :**
  * `✔️ Да` — сбрасывает конфигурацию к значениям по умолчанию;
  * `❌ Нет` — применяет указанные ниже параметры.
* `🌎 Браузер` (`browser`) — имя директории браузера (по умолчанию `Floorp`),  **опционально** .
* `🏙️ Директория закладок` (`bookmarks_folder`) — имя папки закладок, по которой строится отчёт (например, `KDE Store`),  **опционально** .
* `🪪 Кастомный профиль браузера` (`custom_profile`) — имя/путь кастомного профиля Floorp (например, `abcd1234.default-default`),  **опционально** .
* `📁 Название файла репорта` (`custom_report_file`) — базовое имя файла отчёта (без расширения),  **опционально** .

Если поле не передано или пустое, остаётся предыдущее значение из конфигурации.

Ответ:

* JSON с текущей конфигурацией (`browser`, `bookmarks_folder`, `custom_profile`, `custom_report_file`, `is_default`).

##### 2. Очистить от файлов директорию для формирования отчётов

`POST /clear-report-folder`

Эндпоинт безопасно очищает папку для отчётов от файлов.

Поведение:

* проверяет, что директория отчётов существует и является папкой;
* асинхронно удаляет все файлы в этой папке;
* возвращает сводку выполнения очистки в формате JSON.

Примеры ответа:

 - Успешный:
```json
{
  "success": 1,
  "errors": 0,
  "names of errors": []
}
```

 - С ошибками:
```json
{
  "success": 3,
  "errors": 1,
  "names of errors": [
    "Нет прав на удаление файла: docs/locked.txt"
  ]
}
```

##### 3. Получить отчёт по анализу директории закладок

`GET /bookmarks-report`

Эндпоинт принимает данные формы и возвращает отчёт по анализу директории закладок.

Параметры запроса (query):

* `💾 Сохранить файл` (`is_save_file`), **опционально :**
  * `✔️ Да` — отчёт будет сохранён и возвращён как файл;
  * `❌ Нет` — отчёт будет сохранён и возвращён как текст.
* `🏙️ Директория закладок` (`bookmarks_folder`) — **опционально**; если указано, будет использовано вместо значения из конфигурации.


Поведение:

* Если `is_save_file` `=` `✔️ Да`, сервер:
  * сформирует отчёт;
  * сохранит его в папку, указанную в конфиге (`report_folder`, по умолчанию `docs`);
  * вернёт файл отчёта (`text/plain`) как `FileResponse`.
* Если `is_save_file` `=` `❌ Нет`, сервер вернёт отчёт в виде обычного текстового ответа (`PlainTextResponse`).
* Если указанная директория закладок отсутствует в базе, вернётся текстовое сообщение об ошибке.

Пример ответа:

```text
Initial catalog: KDE Store
bookmarks: 0
catalogs: 2
---------------------------------------------------------------------------------------------
Глобальные темы
bookmarks: 3
---------------------------------------------------------------------------------------------
Заставки
bookmarks: 1
```

## Установка и запуск

Проект инициализирован с помощью [uv](https://github.com/astral-sh/uv) и конфигурируется через `pyproject.toml`.

### Требования

- Python 3.14 (см. `.python-version`)
- Установленный `uv`

#### Примечание

Для установки зависимостей проекта через `UV` необходим доступ к PyPI. В некоторых регионах доступ к PyPI напрямую может быть ограничен.

### Загрузка

```bash
git clone https://github.com/VortexOwl/bookmarks-checker.git
```

### Установка зависимостей

```bash
uv sync
```

### Запуск

```bash
uv run start
```

```bash
uv run web
```

```bash
uv run clear
```
